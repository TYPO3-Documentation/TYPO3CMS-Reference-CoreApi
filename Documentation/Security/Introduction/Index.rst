============
Introduction
============

Security is taken very seriously by the `Core`:pn: developers of `TYPO3`:pn: projects
and especially by the members of the official `TYPO3 Security Team`:pn:. It is
also in the interest of system administrators, website owners, editors
and everybody who is responsible for a `TYPO3`:pn: site, to protect the site
and its content against various threats. This chapter describes some
typical risks and advises on how to protect a `TYPO3`:pn: site in order to
ensure it is and stays secure and stable.

This guide also explains how the `TYPO3 Security Team`:pn: deals with
incidents, how security bulletins and security updates are published
and how system administrators should react when their system has been
compromised.

security is a process with ongoing tasks and regular reviews are
essential.

Reporting a security issue
==========================

If you would like to report a security issue in a `TYPO3`:pn: extension or
the `TYPO3 Core`:pn:  system, please report it to the `TYPO3 Security Team`:pn:.
Please refrain from making anything public before an official fix is
released. Read more about the process of incident handling by the
`TYPO3 Security Team`:pn: in the :ref:`next chapter <security-team>`.


Target audience
===============

This chapter is intended for all users of `TYPO3`:pn:, from editors to
system administrators, from `TYPO3`:pn: integrators to software developers.
The Security Guide is an essential lecture for everyone who
works with `TYPO3`:pn: and who is responsible for a publicly accessible
`TYPO3`:pn: site in particular.
